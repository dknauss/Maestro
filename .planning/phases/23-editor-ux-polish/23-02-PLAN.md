<objective>
REVISED 2026-07-05 (user live-iteration decision — see 23-CONTEXT.md §UX-09 geometry).
The original UX-09 "pin the mode/status zone to the admin-menu column" approach was
built, viewed on the running site, and SCRAPPED: it read out-of-sync/misaligned and
was not viable at the menu-column bottom (a lonely pencil in a near-empty 160×45 slot
below the fold).

Then a SECOND user decision (2026-07-05): the bottom-toolbar Exit is REDUNDANT with the
WP Toolbar (admin-bar) toggle, which already flips "Edit Menu" ↔ "Exit" while editing.
Per the plugin's own doctrine (class-admin-bar.php — the toggle is "deliberately hung off
the admin bar"), the Toolbar is the home for entering/exiting. So:

New approach (FINAL): REMOVE the bottom-toolbar Exit and the pencil "Edit Mode" chip.
The admin-bar toggle is the single entry/exit AND the mode indicator — relabel its
editing state "Exit" → "Exit Menu Editor" (the label flip names the mode; no highlight).
CRITICAL: the removed bottom Exit's onExit awaited a pending auto-save before navigating;
the admin-bar toggle is a plain link, so intercept its click while editing to await any
pending save before navigating — no last-change loss. The muted save-status stays in the
toolbar (plan 01). Also fix a reported underline bug on the Reset All control.

Purpose: deliver UX-09's intent (a clear, native edit-mode indicator) with ONE exit in
its wp-native home (the Toolbar), no redundancy, no fragile relocation machinery, and
clear a visible Reset All defect.

Output: the pinned-zone implementation + the redundant bottom Exit removed cleanly; the
admin-bar toggle relabelled "Exit Menu Editor" with a save-flush-on-exit intercept; the
Reset All underline fixed; node:test green.
</objective>
