<objective>
Pin the "Edit Mode" zone (mode chip + save status ONLY) to the admin-menu column
width per UX-09: relocate it into the bottom of #adminmenuwrap, adjacent to core's
Collapse-menu control, seamless with the menu background and set off by a subtle
hairline divider — no plate, no shadow. At <= 782px (where the menu column is
off-canvas) the zone rejoins the full-width bottom toolbar.

Purpose: UX-09 is the one geometry change of the phase. It makes the mode/status
signal read as part of the menu it edits, not as a floating bar element. Everything
else in the toolbar (item controls, rename, Reset All, Exit) stays put.

This plan owns the phase's SECOND pure-logic seam: the 782px relocation gate
(modeZonePlacement), test-first via tests/js (node:test), red before green
(CONTEXT §Methodology TDD boundary). It also carries the live-iteration judgment
work for pinned-zone proportions and how the zone coexists with the Collapse-menu
control — scheduled explicitly as an opus task, outcome recorded in the SUMMARY.

Output: the modeZonePlacement seam + its node:test; relocation wiring in maestro.js;
pinned-zone CSS; and a recorded live-iteration outcome (proportions +
Collapse-menu coexistence decision).
</objective>
